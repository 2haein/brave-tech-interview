__Q1.__ IP주소란 무엇인가요? MAC 주소와 차이점은 무엇인가요?

__A1.__
```
IP 주소: 네트워크 환경에서 컴퓨터간의 통신을 위해 각 컴퓨터에 부여된 네트워크 상 logical 주소 (for 라우팅)
MAC 주소: 네트워크 인터페이스에 할당된 고유한 physical 주소 (for 이더넷 연결)
```


__Q2.__ 다음은 예약된 IP 주소입니다. 어떤 용도로 사용되는 IP주소인지 설명하세요.
```
(A) 0.0.0.0
(B) 10.0.0.1
(C) 224.0.0.1
(D) 192.168.0.0
(E) 255.255.255.255
````
__A2.__
```
(a) 0.0.0.0: 모든 IPv4 주소 지정 방식
(B) 10.0.0.1: 게이트웨이 주소
(C) 224.0.0.1: multicast 전체 호스트 주소
(D) 192.168.0.0: private network
(E) 255.255.255.255: 모든 서브넷 broadcast
````

__Q3.__ IPv4의 A, B, C class에 대해서 설명하세요.
__A3.__
```
클래스는 하나의 IP주소에서 네트워크와 호스트 영역을 나누는 방법이며,
네트워크 크기에 따라 A, B, C, (D, E) 클래스로 나뉜다. 

classA: 하나의 네트워크가 가질 수 있는 호스트 수가 가장 많은 클래스
        1.0.0.0 ~ 126.0.0.0 
classB: 128.0.0.0 ~ 191.255.0.0
classC: 192.0.0.0 c~ 223.255.255.0
```

__Q4.__ AWS EC2 Doc에서 프라이빗 IPv4 주소, 퍼블릭 IPv4 주소, 탄력적 IP 주소에 대한 설명이 나옵니다. (왜 EC2에 앞의 3가지 IP 주소에 대한 설명이 나오는지 생각해보세요) 다른 자료를 참고하여 3가지 IP 주소에 대해 이해한 부분을 작성해보세요.

__A4.__
```
(a) 프라이빗 IPv4
  - 인터넷을 통해 연결할 수 없는 IP 주소이며, 동일 VPC에서 인스턴스 간의 통신을 위해 사용된다. 
(b) 퍼블릭 IPv4
  - 인터넷을 통해 연결할 수 있는 IPv4 주소이며, 인스턴스와 인터넷의 상호 통신을 위해 사용된다.
(c) 탄력적 IP
  - 동적 클라우드 컴퓨팅을 위해 고안된 정적 IPv4 주소로,  인터넷에서 연결 가능한 퍼블릭 IPv4 주소이다. 
  
```
참고: [프라이빗, 퍼블릭 IPv4](https://docs.aws.amazon.com/ko_kr/AWSEC2/latest/UserGuide/using-instance-addressing.html) ,  [탄력적 IP](https://docs.aws.amazon.com/ko_kr/AWSEC2/latest/UserGuide/elastic-ip-addresses-eip.html)
